# Quantitative Aptitude

{: .note }
Quantitative aptitude is a core component of most aptitude tests. Master arithmetic, algebra, geometry, and data interpretation.

## Introduction

Quantitative aptitude refers to the ability to understand and work with numerical information, mathematical concepts, and data analysis. It encompasses a wide range of mathematical topics used in problem-solving.

**Why It Matters:**
- Core component of most aptitude tests
- Measures analytical and logical thinking
- Essential for roles in finance, consulting, and technology
- Companies use it to screen candidates objectively

## Key Areas

| Area | Topics | Importance |
|------|--------|------------|
| Arithmetic | Percentages, profit/loss, ratios | Very High |
| Algebra | Equations, inequalities, quadratics | High |
| Geometry | Mensuration, coordinate geometry | Medium |
| Data Interpretation | Charts, graphs, tables | Very High |
| Number Systems | LCM, HCF, divisibility | High |
| Modern Math | Permutations, combinations, probability | Medium |

## Essential Formulas

### Percentages
- Percentage change = (New Value - Old Value) / Old Value × 100
- Successive percentage change: x + y + (x × y)/100
- If A is x% more than B, then B is [100x/(100+x)]% less than A

### Profit and Loss
- Profit % = (Profit / CP) × 100
- SP = CP × (1 + Profit%/100)
- Discount % = (Discount/MP) × 100

### Time and Work
- If A can do work in x days and B in y days, together: xy/(x+y) days
- Efficiency = 1/Time taken
- Use LCM method for quick calculation

### Time and Distance
- Speed = Distance/Time
- km/h to m/s: Multiply by 5/18
- Average speed (equal distances): 2xy/(x+y)

### Interest
- Simple Interest: P × R × T / 100
- Compound Interest: P(1 + R/100)^T - P
- CI for 2 years: P(R/100)^2

### Number Systems
- LCM(a,b) × HCF(a,b) = a × b
- Divisibility by 3: Sum of digits divisible by 3
- Sum of n natural numbers: n(n+1)/2

### Permutations and Combinations
- Permutations (order matters): nPr = n! / (n-r)!
- Combinations (order doesn't matter): nCr = n! / [r! × (n-r)!]
- Circular permutations: (n-1)!

### Probability
- Probability = Number of favorable outcomes / Total outcomes
- P(A or B) = P(A) + P(B) - P(A and B)
- P(at least one) = 1 - P(none)

## Problem-Solving Approach

1. **Read Question**: Understand what's being asked
2. **Identify Topic**: Recognize the mathematical concept
3. **Recall Formula**: Remember the relevant formula
4. **Plan Solution**: Outline the calculation steps
5. **Execute**: Perform the calculation carefully
6. **Verify**: Check if the answer makes sense

## Best Practices

1. **Master Formulas**: Learn and understand all key formulas
2. **Practice Daily**: Consistent practice builds speed and accuracy
3. **Use Shortcuts**: Learn quick calculation methods
4. **Understand Concepts**: Don't just memorize, understand why
5. **Time Yourself**: Always practice under timed conditions
6. **Start Easy**: Begin with simpler problems, build confidence
7. **Review Mistakes**: Analyze errors to avoid repetition
8. **Use Approximation**: Estimate to eliminate wrong options
9. **Practice Mental Math**: Build calculation speed
10. **Focus on Weaknesses**: Spend extra time on difficult topics

## Common Mistakes to Avoid

| Mistake | Impact | Solution |
|---------|--------|----------|
| Misreading questions | Solves wrong problem | Read carefully, identify what's asked |
| Forgetting unit conversions | Incorrect calculations | Always check units, convert consistently |
| Using wrong formula | Wrong answer | Understand when to use each formula |
| Arithmetic errors | Calculation mistakes | Practice mental math, double-check |
| Not approximating | Wasting time | Use estimation to eliminate options |
