# Online Assessment (OA) Preparation

{: .note }
Online assessments are the first filter in 60%+ of tech hiring processes. Master platforms, time management, and problem-solving strategies.

## Introduction

An Online Assessment (OA) is a digital evaluation tool used by companies to screen candidates. These assessments typically include coding challenges, aptitude tests, behavioral questions, and technical evaluations completed remotely within a time limit.

**Why OAs Matter:**
- First filter in hiring pipeline (60%+ companies use them)
- Performance determines interview advancement
- Tests fundamental skills under time pressure
- Standardizes evaluation across candidates

## OA Platforms Comparison

| Platform | Usage | Features | Time Limit |
|----------|-------|----------|------------|
| HackerRank | 40%+ tech companies | Multiple languages, real-time evaluation | 60-120 min |
| CodeSignal | Startups and mid-size | Standardized scoring (0-850), anti-cheating | 60-75 min |
| HackerEarth | Indian and international | Custom assessments, proctoring | Varies |
| LeetCode | Many companies use similar problems | Extensive library, company-specific | Varies |

## Time Management Strategies

### The 80/20 Rule
- Spend **80% of time** on problems you can solve
- Spend **20% of time** on challenging problems
- Don't get stuck on any single problem

### Time Allocation Framework
| Step | Duration |
|------|----------|
| Read problem | 2-3 minutes |
| Plan solution | 2-3 minutes |
| Implement | 10-15 minutes |
| Test and debug | 5-10 minutes |
| Move to next if stuck | After 20 minutes |

### Question Order Strategy
1. Scan all questions first (2-3 minutes)
2. Identify easy/medium/hard problems
3. Start with confident problems
4. Build momentum and confidence
5. Return to harder problems later

## Common Question Types

### Coding Problems
- Array manipulation and string processing
- Linked list operations
- Tree/graph traversal
- Dynamic programming
- Sorting and searching algorithms

### Aptitude Questions
- Numerical reasoning
- Verbal comprehension
- Logical reasoning
- Data interpretation

### Behavioral Questions
- Situational judgment
- Personality assessment
- Work style preferences

## Key Concepts

| Concept | Definition | OA Impact |
|---------|------------|-----------|
| Time Management | Allocating time efficiently | Maximizes score potential |
| Problem Classification | Identifying difficulty levels | Strategic question selection |
| Platform Familiarity | Comfort with assessment interface | Reduces technical issues |
| Stress Management | Handling pressure during assessment | Maintains performance |
| Code Quality | Writing clean, correct code | Passes test cases |
| Edge Cases | Handling unusual inputs | Prevents wrong answers |

## Best Practices

1. **Start Early**: Begin preparation 4-6 weeks before assessment
2. **Practice on Actual Platform**: Familiarize with interface and submission
3. **Simulate Real Conditions**: Practice with time limits and no resources
4. **Master Fundamentals**: Strong basics solve most problems
5. **Test Thoroughly**: Always test with examples and edge cases
6. **Write Clean Code**: Readable code is easier to debug
7. **Learn from Mistakes**: Analyze failures to improve
8. **Stay Calm**: Manage stress, maintain focus
9. **Have Backup Plans**: Prepare for technical issues
10. **Verify Submission**: Ensure code is submitted successfully

## Common Mistakes to Avoid

| Mistake | Impact | Solution |
|---------|--------|----------|
| Not practicing on platform | Technical issues waste time | Practice on exact platform first |
| Spending too much time on one problem | Miss easier problems | Set time limits, move on if stuck |
| Not testing solutions | Submit incorrect code | Always test with examples |
| Ignoring edge cases | Fails hidden test cases | Consider empty, single, max inputs |
| Poor time management | Run out of time | Allocate time, prioritize problems |

## Quick Reference

### Time Management Rules
- 60-min OA: 4-5 problems → 12-15 min each
- 90-min OA: 5-7 problems → 12-13 min each
- 120-min OA: 6-8 problems → 15 min each

### Common Patterns
- Arrays: Two pointers, sliding window, sorting
- Strings: Hash map, two pointers, sliding window
- Trees: DFS, BFS, recursion
- Graphs: BFS, DFS, union-find
- DP: Memoization, tabulation

### Success Formula
1. Platform familiarity
2. Time management
3. Problem selection
4. Testing approach
5. Code quality
