# Cheat Sheets for Interview Preparation

Cheat sheets are condensed reference guides that capture essential information for quick review before interviews.

## Introduction

Well-crafted cheat sheets allow rapid review of key concepts, formulas, patterns, and strategies before interviews. One page can capture the essence of an entire topic.

## Key Concepts

### Characteristics of Good Cheat Sheets
- Concise: Maximum information in minimum space
- Visual: Uses formatting and layout for quick scanning
- Organized: Logical flow from simple to complex
- Actionable: Can be used immediately during review
- Personalized: Tailored to your specific needs

### Cheat Sheet Categories
- **DSA**: Algorithms, patterns, complexities
- **SQL**: Queries, joins, optimization
- **System Design**: Components, patterns, capacity formulas
- **OOP**: Principles, design patterns
- **Behavioral**: STAR, stories, leadership principles
- **Negotiation**: Scripts, strategies

### Creation Process
1. Select essential content
2. Design layout
3. Apply visual formatting
4. Review and refine

## FAQ

### Q1: How many cheat sheets should I have?
One per major topic area. Keep them focused and concise.

### Q2: How long should a cheat sheet be?
Ideally one page per topic. Two pages maximum.

### Q3: Should I create my own?
Yes. The process of creating them is as valuable as having them.

### Q4: How often should I update them?
Weekly during active preparation. Add new insights and remove outdated information.

### Q5: Should I bring cheat sheets to interviews?
No. They're for preparation, not for use during interviews.

### Q6: What should a DSA cheat sheet include?
Key algorithms, data structure operations, common patterns, time complexities, and quick examples.

### Q7: Should cheat sheets include code?
Minimal pseudocode or key syntax snippets. Focus on concepts and patterns.

### Q8: How do I make cheat sheets visually effective?
Use columns, bullet points, icons, colors, and visual hierarchies.

### Q9: What's the most important cheat sheet?
The one for your weakest area where you need the most review.

### Q10: How do I test if a cheat sheet is effective?
Time yourself scanning it. Key info should be accessible in 5-10 minutes.

## Common Mistakes

1. **Too much content** - Keep it focused and concise
2. **No visual hierarchy** - Use clear organization
3. **Including obvious information** - Focus on what you forget
4. **Never updating** - Stale sheets become useless
5. **Copying without understanding** - Create from memory to force learning
6. **One giant sheet** - Multiple focused sheets are better

## Best Practices

1. Create your own - the process teaches as much as the result
2. Keep it concise - one page per topic maximum
3. Use visual formatting - columns, bullets, icons, colors
4. Focus on essentials - what you forget most
5. Update regularly with new insights
6. Review consistently - daily for weak areas, weekly for all

## Quick Reference

### One-Page Rules
```
□ Maximum one page per topic
□ Focus on what you forget
□ Include key formulas
□ Add quick examples
□ Visually scannable
```

### Review Schedule
```
Daily: Weak areas (5 min)
Weekly: All sheets (30 min)
Before interview: Key sheets (15 min)
```
