# System Design

This category covers comprehensive system design concepts including high-level design (HLD), low-level design (LLD), API design, distributed systems, and cloud computing platforms.

## Topics

| Topic | Description |
|-------|-------------|
| [System Design](system-design) | Core system design principles, HLD/LLD, load balancing, caching, sharding, and common system designs |
| [API Design](api-design) | RESTful API design, HTTP methods, status codes, authentication, pagination, and API documentation |
| [Distributed Systems](distributed-systems) | CAP theorem, consensus algorithms, replication, partitioning, and fault tolerance patterns |
| [Cloud Computing](cloud-computing) | Cloud service models, deployment models, core services, and architecture patterns |
| [AWS](aws) | Amazon Web Services compute, storage, networking, security, and Well-Architected Framework |
| [Azure](azure) | Microsoft Azure compute, storage, databases, networking, and identity management |
| [Google Cloud](google-cloud) | GCP compute, storage, data analytics, networking, and container orchestration |

## Learning Path

1. **Foundation** - Start with System Design fundamentals (HLD/LLD)
2. **API Layer** - Master API Design for service communication
3. **Distributed Concepts** - Understand Distributed Systems theory
4. **Cloud Platforms** - Choose one cloud provider (AWS, Azure, or GCP)
5. **Advanced** - Study multi-cloud and advanced architecture patterns
